# Dhiraj Integration Contract

This contract dictates exactly how output generated by Dhiraj's simulation is mapped into the global causal deterministic state stream.

## 1. Expected Extracted Schema
Dhiraj's layer performs local structural/corrosion logic. Upon interval complete, it must output standard JSON formatted *exactly* by zone keys and deterministic parameter vectors.

```json
{
  "zone_1": {
    "corrosion_rate": 0.02,
    "roughness": 0.3
  },
  "zone_3": {
    "barnacle_density": 0.15
  }
}
```

## 2. Event Translation
The Marine Adapter intercepts this JSON block. It wraps it inside an `ExecutionEvent`:

**Transformation Outcome:**
```python
ExecutionEvent(
    event_id="e54c-...",
    event_type="STATE_UPDATE",
    origin_node="SimulationRunner",
    payload={ "zone_1": { "corrosion_rate": 0.02, ... }, ... },
    timestamp=162391294.0
)
```

## 3. Propagation and Application Update
Once translated:
1. The **Adapter** routes this event to `POST /execute` via the Execution Interface.
2. The **Core Engine** sequences it, dropping conflicts/latency issues, assigning `causal_id=N`.
3. The **Engine** proxies the payload directly back into all replicating nodes.
4. Each node routes the `payload` to their local instances of the Marine Adapter, modifying `ZoneState` fields accurately.
5. Post-apply, a new deterministic SHA-256 hash is confirmed mirroring consensus.
